# GOFAI meets Generative AI

Code, prompts and generated knowledge bases for the paper **"GOFAI meets
Generative AI: Development of Expert Systems by means of Large Language Models"**
(E. C. Garrido-Merchán & C. Puente), *Knowledge-Based Systems*.

The pipeline distils the knowledge of a large language model into a symbolic
**Prolog** knowledge base under a controlled predicate vocabulary, so that it can
be inspected and corrected by human experts and queried by a deterministic
inference engine, combining the recall of LLMs with the precision and
explainability of symbolic systems.

## Repository layout

```
extraction/          LLM -> Prolog extraction scripts (Claude / GPT / Grok)
knowledge_bases/     Generated Prolog knowledge bases, by model
  claude_sonnet_3_7/
  gpt_4_1/
  grok_3/
experiments/         Validation, reasoning and analysis (see experiments/README.md)
  validate_corpus.py     large-scale Wikidata validation + type-aware entity linker
  compute_stats.py       per-model accuracy, Wilson CIs, intra-topic ICC
  run_claude_models.py   multi-model + effort-level extraction/validation harness
  agent_experiment.py    Haiku/Sonnet/Opus tier comparison
  validate_refs.py       mechanical reference verification
  reasoning/rules.pl     Prolog inference-rule layer (multi-hop deduction)
  agent_gen/             knowledge bases generated by Claude Haiku/Sonnet/Opus
  results/               validation outputs (CSV/JSON)
paper/main.pdf       the manuscript
```

## Reproducing the experiments

```bash
cd experiments
python3 -m venv venv && ./venv/bin/pip install "anthropic>=0.69" requests scipy numpy

# Large-scale Wikidata validation across Claude Sonnet 3.7 / GPT-4.1 / Grok 3
./venv/bin/python validate_corpus.py --per-file 14 --per-model 170 --link-eval 150
./venv/bin/python compute_stats.py

# Expert-system reasoning over a generated knowledge base
cd reasoning && swipl -q -g main -t halt rules.pl
```

All Wikidata validation uses the free public Wikidata API (no API key needed)
and is cached locally for reproducibility. Random seeds are fixed.

## API keys

The extraction scripts read credentials from environment variables
(`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `XAI_API_KEY`); no key is stored in the
repository. The reasoning, validation and analysis steps need no API key.
